118 /// \ingroup SlopeLimitersGroup
119 /// \brief A generic Minmod slope limiter
120 ///
121 /// Implements the minmod-based slope limiter from
122 /// \ref cockburn_ref "Cockburn (1999)", Section 2.4.
123 /// Three types of minmod limiter from the reference are implemented:
124 /// \f$\Lambda\Pi^1\f$, \f$\Lambda\Pi^N\f$, and MUSCL.
125 ///
126 /// This minmod limiter has a generic implementation that can work on an
127 /// arbitrary set of tensors. The minmod limiting algorithm is applied to each
128 /// component of each tensor independently. In general, the limiter linearizes
129 /// the tensors on every DG element, each time it is applied; additionally, the
130 /// limiter may reduce the spatial slope of some tensor components if the data
131 /// look like they may contain oscillations.
132 ///
133 /// The key features differentiating the three minmod limiter types are:
134 /// 1. The `Muscl` limiter is the most dissipative; it more aggressively reduces
135 /// the slopes of the data. This limiter may better handle strong shocks, but
136 /// also produces the most broadening of features.
137 /// 2. The `LambdaPiN` limiter is the least aggressive; its "troubled cell"
138 /// detector tries to avoid limiting in DG elements where the data look
139 /// smooth enough. Where `LambdaPiN` is able to avoid limiting, the data are
140 /// _not_ linearized, and the post-limiter data are identical to the
141 /// pre-limiter data.
142 /// 3. The `LambdaPi1` limiter is a middle-ground option between the other two.
143 /// It does not try to avoid limiting as much as `LambdaPiN`, but it allows
144 /// larger slopes in the data than `Muscl`.
145 ///
146 /// For all three types of minmod limiter the "total variation bound in the
147 /// means" (TVBM) correction is implemented, enabling the limiter to avoid
148 /// limiting away smooth extrema in the solution that would otherwise look like
149 /// spurious oscillations. The limiter will not reduce the slope (but will still
150 /// linearize) on elements where the slope is less than \f$m h^2\f$, where
151 /// \f$m\f$ is the TVBM constant and \f$h\f$ is the size of the DG element.
152 ///
153 /// The limiter acts in the `Frame::Logical` coordinates, because in these
154 /// coordinates it is straightforward to formulate the algorithm. This means the
155 /// limiter can operate on generic deformed grids. However, if the grid is too
156 /// strongly deformed, some things can start to break down:
157 /// 1. When an element is deformed so that the Jacobian (from `Frame::Logical`
158 /// to `Frame::Inertial`) varies across the element, then the limiter fails
159 /// to be conservative. In other words, the integral of a tensor `u` over the
160 /// element will change after the limiter activates on `u`. This error is
161 /// typically small.
162 /// 2. When there is a sudden change in the size of the elements (perhaps at an
163 /// h-refinement boundary, or at the boundary between two blocks with very
164 /// different mappings), a smooth solution in `Frame::Inertial` can appear
165 /// to have a kink in `Frame::Logical`. The Minmod implementation includes
166 /// some (untested) tweaks that try to reduce spurious limiter activations
167 /// near these fake kinks.
168 ///
169 /// When an element has multiple neighbors in any direction, an effective mean
170 /// and neighbor size in this direction are computed by averaging over the
171 /// multiple neighbors. This simple generalization of the minmod limiter enables
172 /// it to operate on h-refined grids.
173 ///
174 /// \tparam VolumeDim The number of spatial dimensions.
175 /// \tparam Tags A typelist of tags specifying the tensors to limit.
176 ///
177 /// \anchor cockburn_ref [1] B. Cockburn,
178 /// Discontinuous Galerkin Methods for Convection-Dominated Problems,
179 /// [Springer (1999)](https://doi.org/10.1007/978-3-662-03882-6_2)

250  /// \brief Package data for sending to neighbor elements.
251  ///
252  /// The following quantities are stored in `PackagedData` and communicated
253  /// between neighboring elements:
254  /// - the cell-averaged mean of each tensor component, and
255  /// - the size of the cell along each logical coordinate direction.
256  ///
257  /// \param packaged_data The data package to fill with this element's values.
258  /// \param tensors The tensors to be averaged and packaged.
259  /// \param mesh The mesh on which the tensor values are measured.
260  /// \param element_size The size of the element in inertial coordinates, along
261  /// each dimension of logical coordinates.
262  /// \param orientation_map The orientation of the neighbor
263  void package_data(const gsl::not_null<PackagedData*>& packaged_data,
264  const db::item_type<Tags>&... tensors,
265  const Mesh<VolumeDim>& mesh,
266  const std::array<double, VolumeDim>& element_size,
267  const OrientationMap<VolumeDim>& orientation_map) const
268  noexcept {
269  const auto wrap_compute_means =
270  [&mesh, &packaged_data ](auto tag, const auto& tensor) noexcept {
271  for (size_t i = 0; i < tensor.size(); ++i) {
272  // Compute the mean using the local orientation of the tensor and mesh:
273  // this avoids the work of reorienting the tensor while giving the same
274  // result.
275  get<Minmod_detail::to_tensor_double<decltype(tag)>>(
276  packaged_data->means_)[i] = mean_value(tensor[i], mesh);
277  }
278  return '0';
279  };
280  expand_pack(wrap_compute_means(Tags{}, tensors)...);
281  packaged_data->element_size_ =
282  orientation_map.permute_from_neighbor(element_size);
283  }

291  /// \brief Limits the solution on the element.
292  ///
293  /// For each component of each tensor, the limiter will (in general) linearize
294  /// the data, then possibly reduce its slope, dimension-by-dimension, until it
295  /// no longer looks oscillatory.
296  ///
297  /// \param tensors The tensors to be limited.
298  /// \param element The element on which the tensors to limit live.
299  /// \param mesh The mesh on which the tensor values are measured.
300  /// \param logical_coords The logical coordinates of the mesh gridpoints.
301  /// \param element_size The size of the element, in the inertial coordinates.
302  /// \param neighbor_data The data from each neighbor.
303  ///
304  /// \return whether the limiter modified the solution or not.
305  ///
306  /// \note The return value is false if the limiter knows it has not modified
307  /// the solution. True return values can indicate:
308  /// - The solution was limited to reduce the slope, whether by a large factor
309  /// or by a factor only roundoff away from unity.
310  /// - The solution was linearized but not limited.
311  /// - The solution is identical to the input, if the input was a linear
312  /// function on a higher-order mesh, so that the limiter cannot know that
313  /// the linearization step did not actually modify the data. This is
314  /// somewhat contrived and is unlikely to occur outside of code tests or
315  /// test cases with very clean initial data.
316  bool operator()(
317  const gsl::not_null<std::add_pointer_t<db::item_type<Tags>>>... tensors,
318  const Element<VolumeDim>& element, const Mesh<VolumeDim>& mesh,
319  const tnsr::I<DataVector, VolumeDim, Frame::Logical>& logical_coords,
320  const std::array<double, VolumeDim>& element_size,
321  const std::unordered_map<
322  std::pair<Direction<VolumeDim>, ElementId<VolumeDim>>, PackagedData,
323  boost::hash<std::pair<Direction<VolumeDim>, ElementId<VolumeDim>>>>&
324  neighbor_data) const noexcept {
325  bool limiter_activated = false;
326  const auto wrap_limit_one_tensor = [
327  this, &limiter_activated, &element, &mesh, &logical_coords, &element_size,
328  &neighbor_data
329  ](auto tag, const auto& tensor) noexcept {
330  // Because we hide the types of Tags from limit_one_tensor (we do this so
331  // that its implementation isn't templated on Tags and can be moved out of
332  // this header file), we cannot pass it PackagedData as currently
333  // implemented. So we unpack everything from PackagedData. In the future
334  // we may want a PackagedData type that erases types inherently, as this
335  // would avoid the need for unpacking as done here.
336  //
337  // Get iterators into the local and neighbor tensors, because these are
338  // independent from the structure of the tensor being limited.
339  const auto tensor_begin = make_not_null(tensor->begin());
340  const auto tensor_end = make_not_null(tensor->end());
341  const auto neighbor_tensor_begin = [&neighbor_data]() noexcept {
342  std::unordered_map<
343  std::pair<Direction<VolumeDim>, ElementId<VolumeDim>>,
344  gsl::not_null<const double*>,
345  boost::hash<std::pair<Direction<VolumeDim>, ElementId<VolumeDim>>>>
346  result;
347  for (const auto& neighbor_and_data : neighbor_data) {
348  result.insert(std::make_pair(
349  neighbor_and_data.first,
350  make_not_null(get<Minmod_detail::to_tensor_double<decltype(tag)>>(
351  neighbor_and_data.second.means_)
352  .cbegin())));
353  }
354  return result;
355  }
356  ();
357  const auto neighbor_sizes = [&neighbor_data]() noexcept {
358  std::unordered_map<
359  std::pair<Direction<VolumeDim>, ElementId<VolumeDim>>,
360  std::array<double, VolumeDim>,
361  boost::hash<std::pair<Direction<VolumeDim>, ElementId<VolumeDim>>>>
362  result;
363  for (const auto& neighbor_and_data : neighbor_data) {
364  result.insert(std::make_pair(neighbor_and_data.first,
365  neighbor_and_data.second.element_size_));
366  }
367  return result;
368  }
369  ();
370 
371  limiter_activated =
372  Minmod_detail::limit_one_tensor<VolumeDim>(
373  tensor_begin, tensor_end, minmod_type_, tvbm_constant_, element,
374  mesh, logical_coords, element_size, neighbor_tensor_begin,
375  neighbor_sizes) or
376  limiter_activated;
377  return '0';
378  };
379  expand_pack(wrap_limit_one_tensor(Tags{}, tensors)...);
380  return limiter_activated;
381  }
382 
383  private:
384  MinmodType minmod_type_;
385  double tvbm_constant_;
386 };
